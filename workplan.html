Simple Risk Assessment

RISK ASSESSMENT

Risk #1

 The Development of the system likely will be slowed because project team members have not strong knowledge in PHP prior to this project.

Likelihood Risk

 High Probability of Risk

Potential impact on the project:

 This risk likely will increase the time to complete programming tasks by 50%.

Ways to Address this Risk

 It is very important that time and resources are allocated to up-front training in PHP for the programmers who are used for this project. Adequate training will reduce the initial learning curve for PHP when programming begins. Additionally, outside PHP expertise should be brought in for at least some part of the early programming tasks. This person should be used to provide experiential knowledge to the project team so that PHP-related issues are overcome

Functional Requirements:

WYSIWYG:WYSIWYG implies a user interface that allows the user to view something very similar to the end result while the document is being created.

1. Posting

1.1 The System will enable the Admin to post new Announcements,job offers,announcements,Incoming and Upcoming Events,School Programs.

1.2 The System will allow Admin to reject and accept pending comments.

1.3 The system will notify the users via email for new Upcoming events,Important news and Job offers related to their field.

2. Admin management

2.1 The system will allow Admin to add and remove co-admin.

3. Create Account

3.1 The system will allow Users to login and Register.

3.2 The system will allow Users to Edit Personal information.

4. Search and Browse

4.1 The system will allow Users to browse Any posts.

4.2 The system will allow Users to search for the old and new posts.

4.3 The system will allow other Users to see other Users basic information.

4.4 The system will enable Users to select and read the current or past posts of the school.

4.5 The system will allow Users to comment.

4.6 The System will enable the Guests to search and browse limited posts.

5. Edit posts

5.1 The system will allow Admin to cancel Events.

5.2 The system will allow Admin to Reschedule Events or programs.

5.3 The system will immediately notify users after the rescheduling or cancelation of the event or program.

5.4 The system will display Reschedule Events or programs in the homepage for Guests.

Non-Functional Requirements:

1. Operate

1.1 The system will run on any Web browsers.

1.2 The system will run on any Devices.

1.2 The system is Mobile friendly (web responsiveness).

2. Performance

2.1 The browsing speed will kept at an acceptable level.

2.2 The server and the system can support 350 simultaneous users at all other times.

2.3 The system is available 24 hrs per day and 365 days per year.

3.1 Users personal information will be secured and keep private.
